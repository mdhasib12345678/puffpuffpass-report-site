What's done

  • XML sitemap auto-generated from live database (122 URLs today, 25,000+ after migration)
  • robots.txt with sitemap reference and crawler policies
  • Canonical URLs on every page
  • hreflang declarations (es-ES, en-US, x-default) for multi-locale SEO
  • Schema.org Product JSON-LD with offers, availability and pricing
  • Schema.org BreadcrumbList JSON-LD on all category and product pages
  • Schema.org Organization JSON-LD with Spanish business address
  • Speakable specification for voice assistants (Google, Alexa)
  • Open Graph + Twitter Card metadata per page (per-product OG images)
  • Cloudflare cache headers tuned (1 year for hashed assets, stale-while-revalidate for HTML)
  • Image optimisation helper with auto-WebP conversion (activates with Cloudflare Image Resizing)
  • Preconnect + DNS-prefetch for cross-origin resources
  • Custom branded 404 page

Why Performance is at 70 (and how it gets to 90+)

  • Current bottleneck: temporary placeholder product images served from a free third-party host with slow dynamic resizing.
  • Two paths fix it: Phase 5 catalogue migration moves all images to Cloudflare R2 (instant fix), OR enabling Cloudflare Image Resizing on the existing domain (~€5/month, instant fix).
  • Either path delivers Performance 90+ on mobile.
