Technology · Security

Production-grade security from the very first commit.

Customer trust starts with how their data is handled. Every layer of the platform applies modern security best practices.

Transport & headers (live)

  • HTTPS enforced everywhere with HSTS preload (1-year max-age)
  • X-Content-Type-Options: nosniff (blocks MIME sniffing attacks)
  • X-Frame-Options: DENY (clickjacking protection)
  • Referrer-Policy: strict-origin-when-cross-origin
  • Permissions-Policy locked down (geolocation, microphone, camera all denied)

Backend security (live)

  • Strong auto-generated cryptographic secrets for JWT and cookie signing (48-byte base64)
  • CORS strictly allowlisted to known frontend origins (no wildcard)
  • Database connection over TLS (Neon enforces sslmode=require)
  • Admin authenticated via session cookies (httpOnly, sameSite=lax)
  • All secrets stored in hosting platform environment variables — never in source code
  • Daily Postgres backups via Neon point-in-time recovery (free tier covered)

Compliance & customer data (planned)

  • RGPD-compliant cookie banner (required for Spain/EU)
  • Privacy policy + cookie policy pages
  • PCI compliance via Stripe Elements (card numbers never touch our servers)
  • Encrypted customer passwords (bcrypt, work factor 12)
  • Email verification flow before account activation
