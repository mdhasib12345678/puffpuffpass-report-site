How to read this page

The admin panel is the single back-office where you (and any staff you grant access to) control the entire platform. There are 22 separate data areas plus 2 site-wide settings sections. Each one is shown below, with a short note explaining what it does and which part of the frontend it controls.

The pictures are full-page screenshots — you can scroll any of them to see the exact layout an operator sees when managing your platform.

1. Admin sign-in page.

Admin sign-in page. Email + password authentication with rate limiting, audit logging, and optional two-factor authentication. Sessions stored in secure, HTTP-only cookies.

2. Admin home.

Admin home. Every data area on the platform — users, artworks, auctions, orders, content — surfaced in one place. Grouped by purpose so an operator never has to hunt for what they need.

3. Users list.

Users list. Every customer, artist and admin account on the platform. Search by email, filter by role, edit permissions.

4. Artworks catalogue.

Artworks catalogue. The same artworks your customers see on the storefront — managed here in one place. Search, filter, edit, approve, feature, archive.

5. Auctions list.

Auctions list. Real lots with live, upcoming, ended and cancelled status visible at a glance. Each row is a clickable lot showing every bid, the current price and the winner.

6. Bids ledger.

Bids ledger. Every bid placed on every auction — bidder, amount, timestamp, IP. Immutable record for dispute resolution.

7. Orders.

Orders. Every paid order with line items, payment status, fulfilment status, customer details.

8. Artist profiles.

Artist profiles. Approved artists with their portfolio, biography, sales history and current listings.

9. Artist applications.

Artist applications. New artists submit through the public /apply form on the frontend — applications land here for you to review, approve or reject.

10. Memberships.

Memberships. Paid membership tiers (Pro, Featured) backed by Stripe subscriptions. View status, manage cancellations.

11. Certificates of authenticity.

Certificates of authenticity. Each one is a tamper-evident PDF with the artwork image hash burned in — proof the certificate matches the original art.

12. Events.

Events. Exhibitions, openings, talks. Backed by the same calendar shown on the storefront /events page.

13. Articles & magazine.

Articles & magazine. Rich-text editor with images, videos, embeds. Drafts and published.

14. Media library.

Media library. Every uploaded image with auto-generated thumbnails. Used by artworks, articles, events and certificates.

15. Categories.

Categories. Drives the storefront filters and navigation — change here, it changes everywhere.

16. Newsletter subscribers.

Newsletter subscribers. Email addresses captured from the storefront footer signup. Double opt-in: only confirmed addresses are eligible for sends.

17. Wishlists.

Wishlists. The per-customer "save for later" lists shown in the customer dashboard.

18. Carts.

Carts. Live shopping carts persisted across customer sessions. Abandoned cart recovery hooks ready.

19. Artist portfolios.

Artist portfolios. Curated case-study-style portfolios shown on the /portfolio pages.

20. Case studies.

Case studies. Long-form editorial pieces with multiple sections, images and pull quotes.

21. Careers.

Careers. Job openings displayed on the /careers page. Toggle active/inactive to publish/unpublish.

22. Navigation items.

Navigation items. The storefront header & footer menus are content-managed — you can re-order, rename, or hide items without touching code.

23. Audit log.

Audit log. Every sensitive action — admin logins, role changes, refunds, certificate issuance — recorded with actor, target, IP and timestamp.

24. Site Settings.

Site Settings. Single source of truth for site-wide configuration: contact email, social links, currency, default region.

25. Homepage content.

Homepage content. The hero, featured artists and showcase sections on the storefront homepage — fully content-managed.

26. Your own admin account screen.

Your own admin account screen. Change password, enroll in two-factor authentication, view active sessions.

27. Editing an artwork.

Editing an artwork. Every field shown to customers — title, artist, dimensions, medium, price, certificate flag, status — managed in one form. Slug is auto-filled, status drives whether the piece appears on the storefront.

28. Editing an auction.

Editing an auction. Lot number, start/end times, reserve price, increment, current bid and winner — all in one place. State transitions are validated by the backend to prevent invalid edits.

29. Order detail.

Order detail. Line items, totals, payment status, customer, fulfillment notes. Refunds can be issued from here and immediately flip the order to refunded.

30. User profile.

User profile. Change role, enable/disable account, reset 2FA, view audit history. All changes are audit-logged.

31. One audit log entry expanded.

One audit log entry expanded. Shows actor, action type, target, IP address, user-agent, structured metadata — full forensic trail.
